# Four-Document 100/100 Task List

This checklist turns the July 2026 output review into verifiable product work.
Optional intake fields are enrichment only: leaving them blank must never lower
the audit score or make a generated document look unfinished.

## Executive document

- [x] Use business outcomes and avoid technical/audit vocabulary.
- [x] Reuse the audit engine's health score and explain top limiting risks.
- [x] Provide prioritized next actions with severity, effort, and finding ID.
- [x] Include a compact report-at-a-glance visual summary.
- [x] Remove duplicated risk and recommendation narratives.

## Audit document

- [x] Give every rule-backed finding a stable ID and affected object.
- [x] Show severity, evidence/detail, impact, fix, benefit, and effort.
- [x] Publish transparent checks-run/pass/fail/suppressed category ledgers.
- [x] Keep missing optional business context outside technical scoring.
- [x] Provide prioritized remediation and executable fix snippets.

## Technical document

- [x] Label provenance as extracted, AI-inferred, or human-provided.
- [x] Document cardinality, filter direction, active state, and lineage.
- [x] Include DAX dependencies, complexity, repeated patterns, and risks.
- [x] Analyse schema shape, cardinality signals, unused assets, naming, and calculated columns.
- [x] Include refresh, security, RLS, deployment, and access details when available.
- [x] Use stable searchable anchors for report and model objects.
- [x] Present absent optional intake as neutral unavailable context.

## Business user guide

- [x] Organize guidance by real report page and user workflow.
- [x] Explain purpose, KPIs, filters, navigation, drill-through, and decisions supported.
- [x] Explain visuals in plain language and include common scenarios when grounded.
- [x] Include getting-started guidance, FAQ/troubleshooting, and limitations.
- [x] Exclude technical vocabulary and omit unsupported sections.

## Shared quality and consistency

- [x] Reuse one glossary and shared report facts across documents.
- [x] Cross-link findings, recommendations, pages, measures, and evidence with stable IDs.
- [x] Use one responsive, accessible visual shell and consistent severity styling.
- [x] Guard against unsupported claims, contradictions, duplicate IDs, dead links, and leaked placeholders.
- [x] Regenerate golden snapshots after the neutral optional-context presentation change.
- [x] Run the complete automated suite and inspect all four regenerated HTML outputs.
